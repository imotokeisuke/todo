# 台帳 — タスク管理アプリ

アイテム（管理したいトピック）ごとに TO（やること）と DO（完了したこと）を記録できる、個人用のシンプルなタスク管理アプリです。iPhone のホーム画面に追加して、ネイティブアプリのように使うことを想定しています。

- データはお使いのブラウザ内（localStorage）に保存されます。外部サーバーには一切送信されません。
- 1人で使うことを想定しており、ログイン機能やデータ共有機能はありません。

## ファイル構成

```
index.html      … アプリ本体（これがトップページになります）
manifest.json   … ホーム画面追加時のアプリ名・アイコン設定
icon-180.png    … iPhoneホーム画面用アイコン
icon-192.png    … PWA用アイコン（小）
icon-512.png    … PWA用アイコン（大）
```

## GitHub にアップロードして公開する手順（ブラウザだけでOK）

コマンド操作は不要です。GitHub のWeb画面だけで完結します。

1. [github.com](https://github.com) にログインする（アカウントがなければ無料登録）。
2. 右上の「+」→「New repository」をクリック。
3. Repository name に好きな名前（例: `my-ledger-app`）を入力し、Public を選択して「Create repository」をクリック。
4. 作成されたリポジトリ画面で「uploading an existing file」（または「Add file」→「Upload files」）をクリック。
5. このフォルダの中身（`index.html`、`manifest.json`、`icon-180.png`、`icon-192.png`、`icon-512.png`）を**すべて**ドラッグ＆ドロップする。
6. 下部の「Commit changes」をクリックしてアップロードを確定する。
7. リポジトリ画面上部の「Settings」タブを開く。
8. 左メニューの「Pages」をクリック。
9. 「Build and deployment」の「Branch」を `main`（またはデフォルトブランチ）、フォルダを `/ (root)` に設定し、「Save」をクリック。
10. 数十秒〜数分待つと、ページ上部に公開URLが表示されます（例: `https://ユーザー名.github.io/my-ledger-app/`）。このURLがあなたのアプリのアドレスです。

## iPhoneでアプリとして使う手順

1. 上記で発行された公開URLを iPhone の **Safari** で開く（Safariで開くことが重要です）。
2. 画面下部の共有ボタン（□に↑）をタップ。
3. 「ホーム画面に追加」をタップし、名前を確認して「追加」をタップ。
4. ホーム画面にアイコンが追加され、以降はアイコンをタップするだけでアプリのように起動します（アドレスバーなどが表示されないフルスクリーン表示になります）。

公開URL経由で開くため、以前のようにファイルを直接開く場合とは異なり、ブラウザの保存機能（localStorage）が安定して働き、データもきちんと保持されます。

## 更新したいとき

`index.html` を修正した後、GitHubのリポジトリ画面で該当ファイルを開き、鉛筆アイコン（Edit）から編集するか、再度「Upload files」で同じファイル名を上書きアップロードしてください。「Commit changes」をすると、公開URLの内容も自動的に更新されます（反映まで1分程度かかることがあります）。

## プライバシーについて

このリポジトリを Public（公開）で作成した場合、コード自体は誰でも閲覧可能になりますが、アプリ内に入力したタスクやメモなどのデータは各利用者のブラウザ内にのみ保存されるため、他人に見られることはありません。他人に見られたくない場合は、リポジトリを Private に設定することもできますが、その場合は GitHub Pages の公開に有料プラン（GitHub Pro等）が必要になる点にご注意ください。
